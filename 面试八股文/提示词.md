## 提示词

请你基于我提供的两个 PDF 文件，重新整理并编写一套高质量的 C/C++ 面试八股文 Markdown 文档。

### 一、输入文件说明

1. `面试/C++方向面试题汇总-无答案版本.pdf`
   这是**主输入文件**，其中包含完整的面试题目录和题目列表。你必须以该文件中的题目为准进行整理和作答。
2. `面试/Cpp面试八股文.pdf`
   这是**参考答案文件**，其中包含部分题目的已有答案。但该文件中的答案存在较多错误、不严谨、不完整或表述混乱的问题，因此只能作为参考，不能直接照搬。

### 二、核心任务

请根据 `C++方向面试题汇总-无答案版本.pdf` 中的题目，重新编写一套规范、准确、系统、适合面试复习的 Markdown 文档。

最终产物命名为：

```text
面试八股文.md
```

但由于题目数量很多，你需要先评估内容规模。如果单个 Markdown 文件过长、不利于阅读和维护，请采用**多文件拆分方案**，例如：

```text
面试八股文/
├── README.md
├── 01_C-C++基础.md
├── 02_Linux.md
├── 03_网络.md
├── 04_数据结构与算法.md
├── 05_数据库.md
├── 06_设计模式.md
├── 07_项目.md
└── 09_面试真题.md
```

其中 `README.md` 作为总目录，链接到各个分文件。

### 三、题目处理要求

1. 必须覆盖 `C++方向面试题汇总-无答案版本.pdf` 中的所有题目。
2. 需要忽略以下部分：
   - `8. 开放性问题`
   - `8. 开放性问题` 下的 `1. 项目相关`
3. 其他章节中的所有题目都需要回答，包括但不限于：
   - C/C++基础
   - 面向对象
   - STL
   - Linux
   - 进程和线程
   - 网络
   - 数据结构与算法
   - 数据库
   - Redis
   - 设计模式
   - 项目
   - 面试真题
4. 如果原题本身存在错误、歧义、错别字、不严谨表述或不合理前提，你需要先修正题目，再给出答案。
5. 如果 `Cpp面试八股文.pdf` 中已有答案：
   - 正确且完整的，可以参考其思路；
   - 错误的，必须纠正；
   - 不完整的，必须补充；
   - 表述不清的，必须重写；
   - 过时的，需要更新为现代 C++ / Linux / 网络 / 数据库的常见面试口径。

### 四、答案质量要求

每道题的答案需要做到：

1. **准确**：避免错误概念、过度简化和模糊说法。
2. **完整**：覆盖面试中常见追问点。
3. **结构清晰**：优先使用分点、表格、代码块、流程图等形式。
4. **适合背诵与复习**：答案要有重点，不要写成冗长教材。
5. **适合面试表达**：尽量给出“面试时可以这样回答”的版本。
6. **必要时补充代码示例**，尤其是：
   - C/C++ 语法题
   - STL 题
   - Linux 系统编程题
   - 网络编程题
   - 数据结构与算法题
   - 设计模式题
7. **必要时补充图示或 Mermaid 图**，例如：
   - TCP 三次握手 / 四次挥手
   - 进程状态转换
   - epoll 工作流程
   - 虚函数表
   - 红黑树 / B+树
   - Reactor / Proactor
8. 可以适当使用以下 Markdown 标识块丰富内容：

```markdown
> [!NOTE]
> 补充说明

> [!TIP]
> 面试技巧

> [!CAUTION]
> 常见误区

> [!IMPORTANT]
> 重点结论
```

### 五、Markdown 格式要求

整体格式必须规范。

#### 一级标题格式

一级标题对应大章节，例如：

```markdown
# C/C++基础
# Linux
# 网络
# 数据结构与算法
# 数据库
# 设计模式
# 项目
# 面试真题
```

#### 二级标题格式

二级标题用于章节下的子分类，例如：

```markdown
## 面向对象
## STL
## 进程和线程
## 链表
## 树
## 排序
## 查找
## MySQL
## Redis
## 网盘项目
## 搜索引擎项目
## Http 项目
```

#### 三级标题格式

三级标题用于具体题目，格式必须统一：

```markdown
### 1. 描述 C++ 程序的内存由哪几个部分组成，每个区域分别有什么作用和特点？

答案内容……
```

同一小节内题号从 1 开始顺序编号。

如果原 PDF 中题号混乱、重复或结构不清晰，请你重新整理编号，但不要遗漏题目。

### 六、推荐答案模板

每道题建议使用如下结构：

```markdown
### 题号. 题目标题

#### 简洁回答

用 3～6 句话给出面试时可以直接回答的版本。

#### 详细解释

分点解释核心概念、原理、适用场景和注意事项。

#### 示例代码

如果该题适合用代码说明，请给出简洁、正确、可编译或接近可编译的 C/C++ 示例。

#### 常见追问

- 追问 1：……
- 追问 2：……

> [!CAUTION]
> 常见误区说明。
```

如果某些题目不适合完整模板，可以适当简化，但必须保证答案质量。

### 七、分阶段执行要求

由于题目数量较多，请不要一次性粗略生成所有内容。请按以下阶段完成：

#### 阶段 1：分析与规划

先完成以下内容：

1. 解析两个 PDF 的结构。
2. 列出需要处理的章节。
3. 明确哪些章节需要忽略。
4. 统计大致题量。
5. 判断是否需要拆分为多个 Markdown 文件。
6. 给出最终文件组织方案。
7. 给出每个文件的大纲。

#### 阶段 2：生成目录与文件骨架

根据阶段 1 的规划，先生成：

1. `README.md`
2. 各章节 Markdown 文件的目录结构
3. 每个章节内的标题层级和题目列表

此阶段只需要生成骨架，不需要完整答案。

#### 阶段 3：逐章节补全答案

按以下顺序逐章补全：

1. C/C++基础
2. 面向对象
3. STL
4. Linux
5. 进程和线程
6. 网络
7. 数据结构与算法
8. 数据库
9. Redis
10. 设计模式
11. 项目
12. 面试真题

每完成一个章节后，需要进行一次自检：

- 是否遗漏题目
- 是否修正了错误题目
- 是否存在明显概念错误
- Markdown 层级是否规范
- 代码块是否格式正确

#### 阶段 4：整体质量检查

所有章节完成后，请进行最终检查：

1. 检查所有题目是否覆盖。
2. 检查开放性问题是否已忽略。
3. 检查标题层级是否符合要求。
4. 检查是否存在重复题目。
5. 检查答案是否存在明显错误。
6. 检查是否需要增加图示、Note、Tip、Caution、Important。
7. 输出最终文件清单和完成情况说明。

1. 使用中文编写。
2. 保持专业、清晰、适合面试复习。
3. 避免口水话。
4. 不要简单复制 PDF 原答案。
5. 不要因为题目多就省略答案。
6. 对于重复题目，可以合并回答，但必须在目录或题目下说明“与某题重复，答案见某章节”。
7. 对于存在多个面试口径的问题，要给出主流答案，并说明边界条件。
8. 对 C++ 标准相关问题，优先采用 C++11 及之后的现代 C++ 视角。
